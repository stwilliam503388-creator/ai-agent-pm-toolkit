# AI Agent PM 6层知识体系详解

> 整个12周学习的知识骨架。每一层回答了PM需要掌握的核心问题。

---

## 第1层：LLM基础

**核心问题**：大模型是怎么工作的？

### PM必须知道的
- **Transformer架构**：为什么自注意力机制是革命性的？
- **Tokenization**：中文和英文的Token差异（中文一个字2-3 tokens vs 英文一个词1-2 tokens）
- **上下文窗口**：1M token听起来很大，但成本多高？延迟多大？
- **推理参数**：Temperature/Top-P/Top-K如何影响输出的创造性和一致性？

### PM决策影响
- 选模型时看什么参数？（不只是大小，还有上下文窗口、速度、成本）
- 多语言产品需要考虑Tokenization差异导致的成本差异

---

## 第2层：Agent核心

**核心问题**：Agent和普通LLM有什么区别？

### PM必须知道的
- **ReAct范式**：Reason(思考) + Act(行动)。Agent能"做"而不只是"说"
- **Tool Use**：Agent如何调用外部工具？函数调用的机制是什么？
- **Planning**：Agent如何拆解复杂任务？Plan-and-Execute vs ReAct
- **Memory**：三层记忆架构（短期/长期/工作）

### PM决策影响
- 你的产品需要Agent吗？还是普通LLM就够了？
- 单Agent还是多Agent？决策依据是什么？

---

## 第3层：工具与协议

**核心问题**：Agent如何与外部世界交互？

### PM必须知道的
- **Function Calling**：模型层级，输出JSON表达调用意图
- **MCP协议**：标准化工具发现与调用（Anthropic）
- **A2A协议**：Agent间通信协议（Google）
- **Plugin/Skill系统**：如何封装可复用的能力单元？

### PM决策影响
- 什么时候用MCP，什么时候用Function Calling？
- 工具的安全边界怎么设计？（只读/写入/执行分级）
- 未来切换到新框架时，工具的迁移成本？

---

## 第4层：RAG与知识

**核心问题**：Agent如何"知道"它不知道的事？

### PM必须知道的
- **检索流程**：查询 -> Embedding -> 向量检索 -> Reranker -> 注入Prompt
- **分块策略**：5种策略的适用场景（固定/递归/语义/父子/结构感知）
- **向量数据库**：Pinecone/Milvus/pgvector/Chroma选型
- **Embedding模型**：bge-m3/text-embedding-3-large/jina对比
- **评估**：RAGAS 4个指标（Faithfulness/Answer Relevancy/Context Precision/Context Recall）

### PM决策影响
- 什么场景需要RAG？（知识频繁更新、需要溯源）
- 什么场景不需要？（确定性强、延迟极度敏感）
- 分块策略直接影响检索质量，是最容易被忽略的PM决策点

---

## 第5层：评估与安全

**核心问题**：Agent上线前必须通过什么关卡？

### PM必须知道的
- **评估体系**：成功率、退化率、Token趋势、用户修正率
- **评估方法**：标准化任务 + 多轮重复 + 自动化+人工混合
- **安全威胁**：Prompt注入、工具滥用、角色扮演、上下文泄露
- **防御策略**：输入过滤、工具分级、用户确认、审计日志
- **监控工具**：RagaAI-Catalyst、openlit

### PM决策影响
- 上线前的评估标准是多少？（成功率>90%？修正率<15%？）
- 什么操作需要用户二次确认？什么可以自动执行？
- 告警策略怎么分级？（P0/P1/P2/P3）

---

## 第6层：产品与商业

**核心问题**：如何把技术能力转化为产品价值？

### PM必须知道的
- **PRD设计**：Agent PRD和传统PRD的区别（评估替代功能清单、不确定性管理）
- **场景选择**：什么场景适合Agent？有明确目标但路径不确定=适合
- **设计哲学**：5种设计哲学（OS/导演/触手/记忆/专才）
- **成本模型**：Token定价、ROI分析、模型选型决策树
- **面试策略**：8家公司面经、Top 10高频考点、STAR故事

### PM决策影响
- 这是一个PM的核心战场。前5层知识服务于这一层的决策。
- 能写PRD的PM很多，能用数据论证Agent选型的PM很少。
- 懂Agent产品的PM稀缺，懂+能做+能讲的更稀缺。

---

## 6层之间的关系

```
第6层 产品与商业 ← 所有知识的最终落地点
  ↑
第5层 评估与安全 ← 上线前的最后关卡
  ↑
第4层 RAG与知识 ← Agent的知识来源
  ↑
第3层 工具与协议 ← Agent的行动能力
  ↑
第2层 Agent核心 ← Agent的思维方式
  ↑
第1层 LLM基础 ← 一切的地基
```

每层往下是技术基础，往上是产品价值。PM的价值在于：把第1-5层的技术理解，转化为第6层的产品决策。
