## 核心概念

## 1. MCP的核心价值：标准化工具发现与调用

MCP之前，每个Agent框架有自己的工具调用方式。LangChain有Tool类，AutoGPT有Plugin。MCP定义一个标准协议，让任何Agent用同样方式发现和调用任何工具。

PM视角类比：MCP之于AI工具 = USB-C之于硬件接口。

核心三要素：Tool（工具定义）、Resource（数据资源）、Prompt（提示词模板）。

## 2. Function Calling vs MCP vs Skill：三层架构

2026年Agent面试第一高频考点：

| 层级 | 概念 | 做什么 | 类比 |
|------|------|--------|------|
| 语言层 | Function Calling | 模型输出JSON表达调用意图 | 大脑说拿杯子 |
| 工具箱层 | MCP | 标准化工具发现与调用 | 手知道杯子在哪 |
| 操作手册层 | Skill | 封装完整业务流程 | 调酒SOP手册 |

三层不竞争，是分工：FC做底层通信 -> MCP做工具注册 -> Skill做业务封装。

## 3. MCP的PM决策清单

**用MCP**：跨框架兼容、工具多且变化频繁、需要权限控制

**不用MCP**：只有1-2个工具且不扩展、延迟极度敏感、封闭系统

**安全边界设计**：
- 工具分级：只读 vs 写入 vs 执行
- 用户确认：高风险操作（删除/支付）需二次确认
- 调用频率限制：防止工具滥用

## 面试要点

**Q1：怎么说服技术团队采用MCP而不是自建协议？**
- 生态优势：已有200+ MCP Server可复用
- 维护成本：社区维护 vs 自己维护
- 招聘优势：候选人更可能熟悉MCP
- 承认缺点：协议还在快速迭代，可能有breaking changes

**Q2：MCP和A2A有什么区别？**
- MCP：Agent<->工具（垂直），解决怎么调用工具
- A2A：Agent<->Agent（水平），解决怎么协作
- 不合并原因：场景不同。工具调用是单次同步的，Agent协作是持续异步的
