---

## 核心概念

## 1. 四个模块的交叉关系

| 关系 | 具体表现 |
|------|---------|
| MCP x 成本 | MCP工具调用消耗Token，每次调用都计费 |
| A2A x 成本 | Agent间通信也消耗Token，多Agent=多倍成本 |
| 多模态 x 成本 | 图片/视频分析的Token消耗远大于纯文本 |
| MCP x A2A | MCP管工具层，A2A管Agent层，可以共存 |

## 2. 系统思维

作为PM，你不是管单个技术，而是管一个系统。系统中的每个组件相互影响：
- 加了多模态能力 -> 成本翻倍 -> 需要重新评估ROI
- 引入了A2A -> Agent变多 -> 监控复杂度指数上升
- 换了MCP工具 -> 需要重新做安全审计

## 3. 建立你的决策框架

面对任何一个技术决策时，从四个维度评估：
1. 能力提升（能多做什么）
2. 成本增加（多花多少钱）
3. 风险引入（可能出什么问题）
4. 维护负担（谁来维护）
