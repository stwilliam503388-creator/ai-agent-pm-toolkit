# Day 10 — 成本经济学 - Token定价与模型选型

## 今日目标

掌握Token定价模型，建立模型选型决策树，能估算Agent产品的运营成本。

---

## 核心概念

## 1. Token定价模型

理解三个价格概念：

| 概念 | 含义 | 举例（GPT-4o） |
|------|------|---------------|
| Input Price | 输入Token单价 | $2.50/1M tokens |
| Output Price | 输出Token单价 | $10.00/1M tokens |
| Cache Hit | 缓存命中的输入Token | $1.25/1M tokens（半价） |

关键洞察：输出比输入贵3-4倍。设计Agent时，让模型输出尽量短（让工具做重活，模型只说结论）。

## 2. 模型选型决策树

你需要一个快速决策框架：

```
任务延迟敏感？
  是 -> 小模型（GPT-4o-mini / Claude Haiku）
  否 -> 需要复杂推理？
          是 -> 大模型（GPT-4o / Claude Opus）
          否 -> 中等模型（GPT-4o / Claude Sonnet）

任务需要多模态？
  是 -> 排除纯文本模型（DeepSeek等）

预算紧张？
  是 -> 考虑开源模型（DeepSeek / Llama）+ 自部署
```

## 3. 真实成本案例

假设一个客服Agent，日均1000次对话，每次平均：
- 输入：2000 tokens
- 输出：500 tokens
- 模型：GPT-4o

日成本 = 1000 * (2000*$2.50 + 500*$10.00) / 1M = $10/天

月成本约$300。如果优化Prompt让输出从500降到300 tokens，月省$60。

## 面试要点

**Q1：如何估算一个新Agent产品的运营成本？**
- 先估算日均调用量（用户数*人均调用次数）
- 再估算每次平均Token消耗（Input+Output）
- 乘以模型单价 = 日成本
- 加30%buffer（应对峰值和Prompt膨胀）
- 面试时能当场估算，加分巨大

**Q2：怎么跟老板解释Agent的Token成本？**
- 不要讲Token，讲单次交互成本
- 对比人工成本：Agent一句话$0.01 vs 人工客服一句话$0.50
- 展示优化空间：Prompt压缩、缓存、模型降级等
