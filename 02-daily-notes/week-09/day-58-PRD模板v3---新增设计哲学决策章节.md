## 核心概念

## 1. v3的新增内容

在v2基础上增加：
```
## 设计哲学决策
- 本产品采用的设计哲学：[OS派/导演派/触手派/记忆派/专才派/混合]
- 选择理由：[基于用户场景和技术约束的分析]
- 排除的哲学及原因：[为什么不用其他四种]
- 哲学对产品的影响：
  - 架构层面：[单Agent vs 多Agent]
  - 记忆层面：[是否需要长期记忆]
  - 工具层面：[工具粒度设计]
  - 扩展层面：[未来切换哲学的成本]
```

## 2. 为什么PRD需要设计哲学

大多数PRD只写做什么和怎么做，不写为什么这样设计。
设计哲学章节回答的就是这个为什么。它让技术团队理解你选型的底层逻辑，而不是机械执行。
