# Day 17 — 混合搜索 - Dense加Sparse加Reranker

## 今日目标

理解混合搜索的必要性，掌握三种搜索方式的特性，知道Reranker在架构中的位置。

---

## 核心概念

## 1. 为什么需要混合搜索

纯语义搜索(Dense)：不懂专有名词（搜BMW找不到宝马）
纯关键词搜索(Sparse)：不懂同义词（搜汽车找不到轿车）

混合搜索 = Dense(语义) + Sparse(关键词)，取两者之长。

## 2. 三种搜索对比

| 类型 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| Dense | 向量相似度 | 理解语义 | 专有名词弱 |
| Sparse | BM25/TF-IDF | 精确匹配强 | 不懂同义词 |
| Hybrid | Dense+Sparse融合 | 两者之长 | 实现复杂 |

## 3. Reranker的作用

初检索(1000条->100条，快速粗筛) -> Reranker(100条->10条，精细排序)

Reranker比初检索模型更精确但更慢，只对Top-K结果排序。典型如BGE-Reranker、Cohere Rerank。

## 4. 典型混合搜索架构

```
用户查询 -> Dense检索(1000条) + Sparse检索(1000条)
  -> RRF融合去重(200条) -> Reranker重排序(10条) -> 注入Prompt
```

## 面试要点

**Q1：PM需要知道Reranker的技术细节吗？**
- 不需要知道怎么训练，但要知道：它比向量检索慢10-100倍、中文和英文Reranker不能混用、它是额外成本

**Q2：产品真的需要混合搜索吗？**
- 用户主要搜自然语言 -> Dense够
- 用户经常搜专有名词/ID -> 需要Sparse
- 检索结果经常不够精准 -> 需要Reranker
